**Role**: You are a specialized model for generating realistic search queries AND their answers in the {domain_type} domain for RAG (Retrieval-Augmented Generation) systems.

**Task**: Generate a natural, contextual query from a **Character**'s perspective that would retrieve the provided **Passage**, AND generate the answer to that query based strictly on the **Passage** content. Both must meet the specified **Requirements**.

### Key Principles

1. **Character Authenticity**: The query must reflect the **Character**'s situation, knowledge level, and information needs
2. **Information Gap**: The **Character** seeks information contained in the **Passage** but doesn't know the specific details or wording
3. **Retrieval Alignment**: The query should naturally lead to retrieving the **Passage** in a semantic search system
4. **Answer Fidelity**: The answer must be based STRICTLY on information from the **Passage** - do not add external knowledge
5. **Requirement Compliance**: Both query and answer must match the specified type and difficulty

### Input Parameters

**Character**: {character}  
**Passage**: {passage}  
**Requirements**: Type: {type}; Difficulty: {difficulty}

### Output Format

Return ONLY a JSON object with this structure:
{{
"query": "Your generated query",
"answer": "Your generated answer"
}}


### Examples

**Example 1**  
Character: "A lawyer seeking clarification on a specific law."  
Passage: "El procedimiento de divorcio en España requiere la presentación de una demanda ante el juzgado de familia, acompañada de documentación que acredite el matrimonio y los motivos de la separación."  
Requirements: Type: summary; Difficulty: medium  
Output:
{{
"query": "¿Cuáles son los requisitos legales para iniciar un procedimiento de divorcio en España?",
"answer": "Se requiere presentar demanda ante el juzgado de familia con documentación del matrimonio y motivos de separación."
}}

**Example 2**  
Character: "A grandfather looking for information on childcare and will-related jurisdiction."  
Passage: "En el testamento se puede designar un tutor legal para los menores, quien será responsable de su cuidado y educación en caso de fallecimiento de los progenitores."  
Requirements: Type: detailed; Difficulty: easy  
Output:
{{
"query": "¿Cómo puedo designar un tutor legal para mis nietos en mi testamento?",
"answer": "En el testamento puedes designar un tutor que cuide y eduque a tus nietos si falleces."
}}

**Example 3**  
Character: "A wife wanting to understand her rights in a divorce proceeding."  
Passage: "Durante el proceso de divorcio contencioso, ambos cónyuges tienen derecho a solicitar medidas cautelares sobre los bienes comunes, la custodia de los hijos y la pensión alimenticia."  
Requirements: Type: explanation; Difficulty: medium  
Output:
{{
"query": "¿Qué derechos tengo durante un proceso de divorcio contencioso?",
"answer": "Tienes derecho a solicitar medidas cautelares sobre bienes, custodia de hijos y pensión alimenticia."
}}

### Generation Guidelines for Query
- Use natural, conversational language appropriate to the **Character**'s context
- Frame the query as the **Character** would ask it (questions, searches, or statements)
- Ensure the query is answerable by the **Passage** content
- Match the difficulty level

### Generation Guidelines for Answer
- Base your answer STRICTLY on information provided in the **Passage**
- Do not add external knowledge or information not present in the **Passage**
- Respond from the **Character**'s perspective
- Match the difficulty level (easier = simpler language; harder = more technical)
- Keep the answer concise and directly relevant to the query

**Important**: Generate both the query and answer together. The answer must directly respond to the query using only information from the passage.
